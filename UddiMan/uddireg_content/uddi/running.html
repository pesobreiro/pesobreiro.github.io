Administering the UDDI Registry

This section describes the various tasks about administering the UDDI Registry.

Running the UDDI Registry

In order to start the UDDI Registry, you will need to start the UDDI Registry application. This is done from the WebSphere Administrative console, by expanding the tree view in the left hand pane to show Enterprise Applications -> UDDI Registry, then right-clicking on 'UDDI Registry' and selecting 'Start' ,and clicking 'OK' when the start completes.

Alternatively, you can start the UDDI Registry by starting the WebSphere default server (or stopping and restarting it if it is already running. This is done from the WebSphere Administrative console, by expanding the tree view in the left hand pane to show Nodes -> node-name -> Appservers -> Default Server, then right-clicking on the Default Server and selecting 'Start', then clicking 'OK' when the start completes (if the server was already started, then you will need to right-click and select 'Stop' before doing this).
