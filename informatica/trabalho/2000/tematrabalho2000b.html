Trabalho Prático de Informática

Enunciado do Trabalho:

Uma organização desportiva pretende ter um mecanismo que possibilite de uma forma simplificada o envio de mailings a todas as entidades a que está ligada. Desta forma pretende-se criar uma base de dados que sirva de repositório para todos as entidades.

A Base de dados será constituída pelas seguintes tabelas:

  • Tabela de Zonas, onde serão indicadas as zonas em que estão colocados os contactos.
  • Tabela de Contactos, permite armazenar a informação dos contactos, note-se que no diagrama em anexo existe um campo com o nome foto, básicamente a ideia é armazenar a fotografia do contacto.
  • Tabela de Interesses do Contacto, esta tabela funciona como um elo de ligação entre os Contactos e os Interesses, note-se que isto veio possibilitar a criação de uma relação do tipo muitos para muitos.
  • Tabela de Interesses, permite a introdução dos interesses dos contactos.

Toda a Base de Dados será criada com o objectivo da realização de Mailing's.

A tabela de Interesses serve para indicar quais são os interesses que um contacto têm por ex. numa modalidade Desportiva, Futebol, Basquete, etc... A ideia é realizar uma gestão dos interesses dos contactos de forma a que a realização de um Mailing seja orientado a áreas específicas.

O modelo da base de dados poderá ser baseado na estrutura em anexo, no entanto esta estrutura não é rígida podendo por isso ser alterada.

1. Tarefas

Criar consultas para obter:

  • Os interesses dos contactos por zona?
  • Qual é a zona com mais contactos?
  • Qual é o interesse mais solicitado pelo os contactos?

Criar relatórios para:

  • Listagem de contactos por zona.
  • Listagem de interesses.
  • Listagem de Contactos

Criar formulários para:

Os formulários deverão ser criados para todas as tabelas, excepto para a tabela de InteressesContactos. Também deverá ser criado um ecrã principal que possibilite aceder aos outros formulários.

2. Ligação ao Winword.

Criar um documento no Word como exemplificação desta situação e definir uma consulta já préviamente configurada como a origem de dados do Mailing. Para esta parte do trabalho é necessário entregar o documento do Word base para o Mailing e o resultado da criação do Mailing.

2. Relatório com o máximo de 2 páginas, indicando as dificuldades encontrada e sugestões de melhoria para a Base de Dados.

3. O trabalho deverá ser entregue com as tabelas préviamente carregadas de forma a ser possível a sua análise.
