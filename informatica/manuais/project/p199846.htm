Route a Project File

When you want a project to be reviewed by team members and others, you can route the project file. You can route a project sequentially — so that one person at a time gets it, reviews it, and sends it to the next person — or you can send it to everyone on the routing list simultaneously.

To route a project file

  1. On the File menu, point to Send To, and then click Routing Recipient.
  2. Click Address, hold down CTRL, click the names of the recipients, click To, and then click OK.
  3. To change the order of the recipients, click a name, and then click a Move button.
  4. In the Subject box, type the purpose of the routing message.
  5. In the Message text box, type instructions or other information.
  6. Under Route to recipients, click the delivery option you want.
  7. If you don’t want the file returned to you after the last recipient on the routing list receives the project, clear the Return when done check box.
  8. If you don’t want to be notified each time the file is routed to the next recipient on the routing list, clear the Track status check box.
  9. If you aren’t ready to send the file, click Add Slip to save the routing slip with the project file.
  10. To send the project file to the recipients, click Route.
