By Any Route: Track Workgroup Assignments

Whether you use an e-mail system, an intranet, or the World Wide Web to communicate with team members, you’ll want to exchange core information about your project. For example, you’ll want to send task assignments and receive confirmations and updates.

With Microsoft Project, you can exchange a core set of project information with your workgroup as long as you use an e-mail system or a web-based system. (If you want to route messages, you can do so with an e- mail-system only.) If you’re the workgroup manager and you’re using an e-mail system only, you can receive responses to your TeamAssign, TeamUpdate, and TeamStatus messages only in the inbox of your e-mail program. If you’re using a web-based system only, you can receive responses to your workgroup messages in your WebInbox only.
