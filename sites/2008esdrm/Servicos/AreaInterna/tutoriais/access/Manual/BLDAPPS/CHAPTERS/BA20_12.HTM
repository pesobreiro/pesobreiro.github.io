Making a Replicated Database a Regular Database

After you've converted a database to a Design Master, you can't convert it back to a regular database. However, if you no longer want to use replication, you can create a new, regular database that contains all of the objects and data in your replicated database without the additional system fields, tables, and properties associated with replication.

   To make a replicated database a regular database

1  &nbspCreate and open a new, blank database in Microsoft Access.

2  &nbspOn the File menu, point to Get External Data, and then click Import.

3  &nbspIn the Import dialog box, click the replica that contains the objects that you want in the new, regular database, and then click Import.

4  &nbspIn the Import Objects dialog box, click the objects (except tables) that you want to import into the new database, and then click OK.

5  &nbspIn the new database, create a query for each table in the replica that takes all the data in the replica and puts it into tables in the new database.

6  &nbspFor each table in the new database, create the same indexes and relationships that existed in the replica table.

7  &nbspSave your new database.
