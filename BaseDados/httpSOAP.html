Pedidos SOAP via HTTP

Para o servidor SQL expor um interface SOAP no servidor web é necessário implementar os seguintes passos:

  1. Criar na consola MMC do SQLXML , um nome virtual do tipo SOAP.
  2. Configurar o nome virtual do tipo SOAP, que vai gerar o ficheiro WSDL(Web Services Description Language) que descreve o serviço e as operações que são suportadas, nomeadamente sprocs, user defined functions e templates que o cliente pode pedir.

Notas

  1. A cláusula FOR XML não pode estar definida na sproc, porque a formatação XML é aplicada no lado no cliente.
  2. O processo de configuração também cria um ficheiro com a extensão ssc na pasta que está associada com o nome virtual do tipo SOAP
